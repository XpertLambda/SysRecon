Q: Do I need to provide config.json with sysrecon.exe?
A: YES! The program requires config.json to run. Without it, it will fail.
   If you use the installer, this is included automatically.

Q: What's the difference between the installer and the portable exe?
A: Installer: Professional setup wizard, creates shortcuts, registry entries
   Portable: Just the .exe file, no installation, can run from USB drive
   Both versions work the same once launched.

Q: Can I build on Linux and run on Windows?
A: YES! That's what cross-compilation does. You build on Linux (using MinGW),
   and the resulting sysrecon.exe runs on Windows 10/11.

Q: Why is sysrecon.exe 17 MB but the installer is only 3.9 MB?
A: The installer uses LZMA compression. It's the same 17 MB executable
   compressed into 3.9 MB. When users install, it extracts to 17 MB.

Q: Do users need any runtime libraries?
A: No! sysrecon.exe is statically linked. No Visual C++ Redistributables
   or other dependencies needed.

Q: What if config.json is missing?
A: The program will fail to start. Always include config.json.
   Default config.json is in the project root.

Q: Can I customize the installer?
A: Yes! Edit installer.nsi to change company name, shortcuts, options, etc.
   Then rebuild with ./build-installer.sh
